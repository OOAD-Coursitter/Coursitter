> 1. 原则
```
1. 应用 前后端分离开发
    - 前端只 设计网页 及 所有请求（设置所有请求函数及回调）
    - 后端只 设计接口
    - 前后端 通过 json 交互， 填充网页 及 满足响应
2. REST风格
    - 尽量避免url传参
    - 合理分配全站 url
3. 易用性
    - 合理的取名 使 url 和 api 使用人性化
    - 保留二次开发可能性
4. 维护此设计提交文档
```
